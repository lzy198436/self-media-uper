# self-media-uper：素材目录批量投稿

工具：`smu`（已在 PATH；源码 `/Users/liuzhiyuan/Desktop/self-media-uper`）。
状态记录在 `~/.self-media-uper/state.json`，按素材文件夹名防重复投稿。

## 第一步：确认素材目录（必须）

**永远先和用户确认素材目录，不要猜、不要用历史目录直接开干。**

- 用户没给目录 → 询问：「这次投哪个素材目录？」可以列出 `/Volumes/市场部文件/` 下的候选帮用户选。
- 用户给了模糊名称（如「民诉法」）→ 找到匹配目录后**复述完整路径让用户确认**。
- 目录确认后先跑 `smu scan <目录>` 检查素材完整性（缺封面/文案会有 ⚠️ 提示）。

## 第二步：对账与进度

```bash
smu sync <目录>      # 拉取B站已发布稿件，自动标记已投（手动投的也能识别）
smu status <目录>    # 已投/待投统计 + 下一个待投序号
```

用户问「投到哪了」只需要这两条。

## 第三步：投稿（公开发布前必须向用户确认）

```bash
smu upload <目录> 11            # 投第11个
smu upload <目录> 11-20         # 投11到20
smu upload <目录> --all         # 投全部未投的
smu upload <目录> 11 --private  # 仅自己可见（测试效果用）
smu upload <目录> 11 --dry-run  # 只看将提交的标题/简介/标签，不上传
```

规则：
- **公开投稿是对外发布，执行前必须把「数量 + 起止序号 + 标题前缀」报给用户确认**；用户已明确说了数量/范围（如「投10个」「从11投到20」）则直接执行。
- 批量默认每个间隔 30 秒防风控。**单日公开投稿不要超过 20 个**，用户要求更多时提醒分天投。
- 已投稿的自动跳过；用户明确要求重投才用 `--force`。
- 失败的素材在汇总里列出原因，询问用户是否重试，不要自动无限重试。
- 不要替用户删除任何稿件。

## 第四步：汇报

投稿结束后报告：成功 N 个（列 BV 号）、失败 M 个（列原因）、当前进度（已投/总数）、下一个待投序号。

## 登录态问题

报「未登录」→ 让用户在**自己的终端**（不是 agent 的 shell，需要扫码）运行：`smu login`，选「扫码登录」。
报 cookie 过期 → `smu renew` 可直接代跑。

## 默认投稿参数（可被素材目录下 smu.json 覆盖）

- 标题 = `【2026法考邪修流（民诉）】` + 素材文件夹名（含序号）
- 简介 = `*_文案_B站.txt` 正文（自动去掉 #标签 行）
- 标签 = 话题名 + 法考邪修流/2026法考备考 + 文案里的 #标签（上限10个）
- 封面 = 文件夹内现成的 16:9 主封面 + 4:3 首页封面
- 创作声明 = 含AI生成内容；分区 = 知识；话题 = bilibili法考季

换科目（如刑法）时标题前缀要变：在素材目录放 `smu.json` 写
`{"title_prefix": "【2026法考邪修流（刑法）】"}`，或用 `--title-prefix` 参数。

## 多平台（预留）

素材文件夹里已备好抖音/小红书/视频号文案。这些平台没有公开投稿 API，等
Playwright 适配器实现后用 `--platform douyin` 等切换；当前仅支持 bilibili。
